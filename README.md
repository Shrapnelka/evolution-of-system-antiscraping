# evolution-of-system-antiscraping
## Исследование систем анти-скрейпинга и маскировки трафика
Данный репозиторий содержит практическую часть и исходный код для одноименной научной статьи. Проект посвящен исследованию методов детектирования автоматизированных ботов (WAF) и технологиям эмуляции цифрового следа легитимного клиента на языке Python.

⚠️ Disclaimer (Отказ от ответственности): Все скрипты в данном репозитории созданы исключительно в образовательных и исследовательских целях (Proof of Concept). Авторы не несут ответственности за использование данного кода для несанкционированного сбора данных или нарушения правил использования (Terms of Service) сторонних веб-ресурсов.

Состав проекта:

waf_server.py — Локальный сервер-стенд для эмуляции механизмов WAF (поведенческий анализ, Rate Limiting).

attack.py — Базовый автоматизированный агент. Демонстрирует блокировку (HTTP 429) при превышении лимитов.

scraper_weather.py - Базовый автоматизированный агент. Демонстрирует работу парсинга сайта с погодой и вывод данных о погоде.

scraper_advanced.py — Агент с комплексной маскировкой. Реализует подмену цифрового следа браузера (расширенные HTTP-заголовки) и алгоритм прямого перебора (ID Enumeration) с контролем таймингов.

Запуск:

### Установка и запуск

1. **Клонирование репозитория и установка зависимостей:**
   Убедитесь, что у вас установлен Python 3. Откройте терминал, перейдите в папку проекта и установите необходимые библиотеки:
   ```bash
   pip install -r requirements.txt
   
2. **Запуск тестового стенда (Эмуляция WAF):**
В первом окне терминала запустите локальный сервер. Он поднимет тестовую среду, которая прослушивает входящие запросы и эмулирует механизм Rate Limiting (лимит — 5 запросов в секунду).

```bash
   python3 waf_server.py
  ```
(Внимание: оставьте этот терминал открытым, сервер должен работать в фоновом режиме).

3. **Тестирование базового парсера (Демонстрация блокировки):**
Откройте второе окно терминала и запустите базовый скрипт. Вы увидите в логах сервера и консоли скрипта, как система защиты выдаст ошибку HTTP 429 Too Many Requests после превышения лимита.

```bash
python3 attack.py
```

4. **Запуск скриптов сбора данных:**
Для проверки базового сбора метеоданных выполните:

``` bash
python scraper_weather.py
```
Для запуска агента с расширенной маскировкой и алгоритмом перебора (демонстрация обхода продвинутых WAF):

```bash
python scraper_advanced.py
```
